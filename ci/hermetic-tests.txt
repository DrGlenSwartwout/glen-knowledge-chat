# Hermetic test allowlist -- the PR gate (.github/workflows/tests.yml) runs exactly these.
#
# Why an allowlist and not the whole suite: of ~900 test files, 122 import app.py, which
# reaches out to Pinecone/QBO/Stripe at import time and needs live secrets. Most of the rest
# collect fine but make network calls at runtime. Running everything in CI would be red for
# environmental reasons, not real ones (the local gate diffs the FAILED set against a
# same-day baseline for exactly this reason -- see the repo notes on the no-CI workflow).
#
# So this file lists test files verified to pass with NO secrets and NO network. They are the
# pure business-logic suites -- pricing, packing, ledgers, dispatch/actions, money console --
# which is the highest-value regression surface. Deterministic: green when clean, red only on
# a real regression.
#
# To add a file: run it with no secrets and no doppler, e.g.
#   pytest tests/test_foo.py -q
# It qualifies only if it passes standalone AND still passes when run alongside the whole
# list below.
#
# Blank lines and #-comments are ignored.
