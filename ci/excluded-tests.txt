# Tests EXCLUDED from the CI gate (.github/workflows/tests.yml).
#
# CI runs `pytest tests/` with every file below passed as --ignore, so everything NOT
# listed here is gated -- including test files added in future. That is deliberate: a new
# test is covered by default, and a new test that needs live credentials will fail CI until
# someone consciously adds it here.
#
# Why these are excluded: CI runs with NO real secrets. It sets a DUMMY PINECONE_API_KEY,
# which gets past the Pinecone client CONSTRUCTOR (it only checks the key is non-empty, with
# no network call). That matters because without it 122 files hard-error during COLLECTION,
# which aborts the run for everything else.
#
# It does NOT make app.py usable: app.py issues a live Pinecone call at import, which 401s on
# a fake key. Tests that import app therefore skip themselves (the repo convention is a
# try/except around the import + pytest.skip) rather than run. So the ~3700 gated tests are
# overwhelmingly the pure-logic suites; app-route coverage still needs real credentials.
#
# The files below fail outright rather than skipping cleanly, so they are excluded.
#
# Deliberately NOT solved with a Doppler CI token: the remedy-match dev config carries live
# production money credentials (QuickBooks PROD, Authorize.net, Wise, PayPal, Stripe) plus a
# Cloudflare token and mailbox passwords. Putting that in GitHub Actions would expose
# production financial credentials to every workflow run. The dummy-key approach gets ~3700
# tests gated with zero credentials instead.
#
# To re-check this list (takes ~8 min, safe -- no real credentials are present):
#   PINECONE_API_KEY=dummy-ci-key pytest tests/ -q --continue-on-collection-errors --tb=no -rfE
# Any file with zero FAILED/ERROR lines can be removed from this list.
#
# Blank lines and #-comments are ignored.
